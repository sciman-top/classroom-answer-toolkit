# 外部文档采纳决策表

## 1. 文档定位

本文档用于记录外部参考文档对本项目的采纳结论，避免讨论结论只停留在会话中。

约束：
- 本文档只记录“是否采纳、为何采纳、落到哪里、近期怎么做”。
- 本文档不替代总纲、路线图、实施清单与协议设计。
- 外部文档原文不复制进仓库；仅沉淀对本项目有用的决策。

当前参考来源：
- `docs/research/2026-07-07-product-architecture-review.md`
- `tools/ocr/README.md`

- `docs/archive/工程终态与平台化路线图.md`（仅作历史参考，不再作为当前规划真值）

## 2. 使用原则

- `立即采纳`：与本仓现有方向一致，可直接纳入后续实现与文档维护。
- `近期采纳`：值得吸收，但应排在当前主线之后，以免打断既有改造顺序。
- `条件采纳`：只有在目标范围扩大后才立项，不提前混入当前主链。
- `不采纳`：不符合本项目形态，明确保留边界，避免误迁移。

## 4. 结论

本轮外部文档最值得吸收的是“如何组织长期演进的质量体系”，包括：
- 文档分层
- 契约先行
- 快照收口
- 高价值样例
- 状态分层
- 视觉与作图能力独立验收
- 视觉证据编译优先于模型直答

本轮外部文档最不适合直接迁入的是“QQ 解题机器人运行时求解链路”，包括：
- 双链路路由
- 会话内澄清追问
- 灰度放量
- 在线 provider 门槛

## 5. 外置参考仓分层结论

本轮同步收口“当前外置参考仓是否需要增删”。结论不是增加一批常驻镜像，而是把现有参考仓分成 `active / secondary / candidate / remove-from-active` 四层。

| 参考仓 | 分层结论 | 现阶段定位 | 依据 | 近期动作 |
| --- | --- | --- | --- | --- |
| `MVVM-Samples` | `active` | WPF 交互、ViewModel 组织与桌面应用模式参考 | 当前主产品仍是 Windows-first WPF 工作站 | 保留在活跃参考清单中，供 WPF 结构与交互模式对照 |
| `Wpf.Extensions.Hosting` | `active` | WPF + Generic Host 集成参考 | 当前主链仍基于桌面应用 + Host 编排 | 保留在活跃参考清单中，继续服务宿主与依赖注入设计 |
| `RapidOCR` | `active` | OCR 主链与辅助审阅参考 | 仓内 OCR 文档与现行扫描辅助链仍直接依赖 RapidOCR 路线 | 保留在活跃参考清单中 |
| `marp-cli` | `secondary` | 浏览器导出 PDF 的渲染模式参考 | 当前主渲染链仍是 Playwright/Chromium；`marp-cli` 只提供模式启发 | 从活跃主参考降级为次级参考，不再当作主设计驱动 |
| `Text-Grab` | `remove-from-active` | 旧 OCR/桌面工具参考 | 现有代码、策略文档与 OCR 主链中已无持续依赖关系 | 从当前活跃参考清单移除 |
| `Typst` | `target-after-gates` | 终局主渲染目标，当前仍非运行时默认后端 | ADR 0006 与 D-017 已接受 Typst 主渲染目标，但要求 adapter、parity gate 与 rollback smoke 通过 | 不提前常驻为运行时依赖；先按 renderer contract 和迁移计划推进 |
| `OCRmyPDF` | `candidate` | 归档级扫描 PDF 增强候选 | 仅在大量低质量扫描卷与长期归档场景下才有必要 | 暂不列为常驻外置参考仓，仅保留为候选 |

补充约束：
- 当前不新增新的“默认常驻外置参考仓”。
- 候选项先记录在规划与研究文档中，不提前上升为主线依赖。
- 历史参考仓清单仅保留在 archive 或研究文档中，不反向覆盖 `docs/strategy/` 当前真值面。

## 6. 结论补充

外置参考仓的当前最佳状态是：
- 活跃主参考只保留与现行主链强相关的 `MVVM-Samples`、`Wpf.Extensions.Hosting`、`RapidOCR`
- `marp-cli` 保留但降级
- `Text-Grab` 退出活跃清单
- `Typst` 是终局主渲染目标，但不提前成为当前运行时依赖
- `OCRmyPDF` 只作第二阶段候选，不提前常驻

后续如需更新本表，应同步回看总纲、协议设计、评测目录与工具链入口，避免出现“决策表已变、主文档未变”的漂移。

补充：Typst 已从泛泛候选调整为终局主渲染目标，但这不是当前运行时切换。当前默认交付链仍是 Playwright / Chromium，Typst 必须先通过 renderer contract、parity gate 和 rollback smoke。
