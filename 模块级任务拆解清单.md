# 模块级任务拆解清单

## 1. 文档定位

本文档是 [工程终态与平台化路线图.md](D:/CODE/classroom-answer-toolkit/工程终态与平台化路线图.md) 的执行清单版本。

目标：

- 按模块拆任务
- 明确每个模块的输入、输出、依赖、验收
- 让后续开发可以直接按模块推进

## 2. 执行总原则

- 先治理规则与资产，再治理工具入口
- 先建立统一快照，再重构渲染与交付
- 先做作图题子系统骨架，再做自动化能力
- 先保证课堂可视度和视觉门禁，再追求更高自动化

## 3. 模块一：版本对齐与资产治理

### 目标

将当前最新版规范与资产层完全对齐，消除 `v8.8` 与结构化资产漂移。

### 涉及路径

- `初中物理试卷参考答案生成与渲染交付提示词_v8.8_生产完整版.md`
- `prompts/physics-answer/manifest.json`
- `prompts/physics-answer/config.json`
- `prompts/physics-answer/spec.md`
- `prompts/physics-answer/checklists/acceptance.md`

### 任务

- 任务 1.1：将 `manifest.json` 版本提升到 `v8.8`
- 任务 1.2：将 `config.json` 的 `sourceOfTruth` 指向 `v8.8`
- 任务 1.3：把 `v8.8` 新规则同步到结构化配置
- 任务 1.4：同步 `spec.md`
- 任务 1.5：同步 `acceptance.md`

### 验收

- 根规范与资产层版本一致
- 资产层没有再引用 `v8.7`

## 4. 模块二：平台核心资产层

### 目标

建立 `platform-core` 资产层，承载跨学科稳定不变量。

### 涉及路径

- `prompts/platform-core/manifest.json`
- `prompts/platform-core/rules/`
- `prompts/platform-core/profiles/`
- `prompts/platform-core/migrations/`

### 任务

- 任务 2.1：建立 `platform-core` 目录
- 任务 2.2：拆出交付规则
- 任务 2.3：拆出数学渲染硬门禁
- 任务 2.4：拆出视觉审阅硬门禁
- 任务 2.5：建立默认 profile 基线

### 验收

- 至少有一份平台 manifest
- 至少有一组平台级 rules
- 至少有一组基础 profile

## 19. 结论

这份清单的核心作用，不是把任务列长，而是把后续所有工作都约束到：

- 明确模块
- 明确落点
- 明确依赖
- 明确验收

这样后面无论是你自己做，还是继续交给别的 agent，都会更稳。
