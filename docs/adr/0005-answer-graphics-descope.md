# ADR-0005: 自动作图答案图降级为实验性受控插图

## Status

Accepted

## Date

2026-07-05

## Context

项目早期希望覆盖“基于题图自动生成作图题答案图，并自动插入答案 PDF”的能力，例如：

- 杠杆题自动补画力臂
- 电路实物图自动补连导线
- 数学几何图自动补辅助线

后续复盘后确认，这条链路的真正难点不在渲染，而在：

1. 视觉读图是否正确
2. 学科判断是否正确
3. 图元位置与语义是否一致

一旦错误被画进答案图，误导性会高于普通文字答案错误。

## Decision

不再把“AI 自动基于题图生成作图题答案图”作为本项目的主需求。

保留的仅是实验性底座：

- 用户提供的答案图块插入
- 人工复核后的 placement 记录插入
- 已有图块在 PDF 中的受控排版与追踪

主链交付继续聚焦：

- 题图审查
- 文字答案
- 疑点标注
- PDF 交付质量

## Alternatives Considered

### 继续把自动作图答案图作为主线能力

- Pros: 产品看起来更完整
- Cons: 高风险错误会以更显眼的方式进入最终交付
- Rejected: 与当前视觉可靠性水平不匹配

### 完全删除所有插图相关底座

- Pros: 系统更简单
- Cons: 会失去“插入用户现成答案图/人工图块”的受控承载能力
- Rejected: 受控插图仍然有边际价值

## Consequences

1. `tools/answer-graphics` 保留，但降级为实验性工具链
2. bootstrap / check 主链门禁不再依赖 `tools/answer-graphics`
3. 工作区诊断不再把缺少图块产物视为默认问题
4. 规范、README、路线图不再把自动作图答案图当成交付承诺
