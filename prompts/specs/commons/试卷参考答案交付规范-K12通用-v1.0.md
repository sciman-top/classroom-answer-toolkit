# 试卷参考答案交付规范-K12通用 v1.0

> 适用范围：面向中小学各学段、各学科的试卷参考答案生成、校验、渲染与交付。
> 文件角色：承载跨学科、跨学段都普遍适用的规范。

## 1. 证据优先级

1. 原始 PDF、原始图片、原始截图是最高依据。
2. OCR、自动解析文本、上一轮答案、其他模型答案只能作为辅助。
3. 若原图与 OCR 冲突，以原图为准。
4. 若原图模糊、关键数据缺失、图表不可辨认，不得硬猜，必须标注【疑】。
5. 若疑点只影响局部小问，默认继续完成其他可确定部分，不因单点疑问中断整份交付。

## 2. 交付约束

1. 默认最终交付物至少包含答案 `.md` 与答案 `.pdf`。
2. PDF 中数学公式必须走真实 LaTeX 数学渲染链，不得降级为纯文本近似。
3. 最终输出不得暴露内部审查过程、试错痕迹、工具调用信息或模型自评。
4. 若存在疑点，应输出极简疑点清单，而不是在答案正文中混入长说明。

## 3. 目录与命名

1. `样例交付/` 用于演示、回归、冒烟测试，不与正式交付混放。
2. `正式交付/` 用于真实生产交付。
3. 人类规范采用“源规范 + compiled 生成物”模式，compiled 文件用于转发与工具引用。

## 4. 可读性与答题风格

1. 答案应优先服务课堂展示、打印分发与复核，不应写成冗长讲解稿。
2. 应优先输出结论、关键式、必要前提，不堆叠无新增价值的解释句。
3. 若用户明确要求补分析，应只补与结论直接相关的必要推导。

## 5. 人工复核原则

1. 高风险内容应允许进入人工复核，而不是强行全自动放行。
2. 复核结论应可追踪到具体证据，而不是停留在主观描述。
3. 仅当疑点会阻断整份交付的主要判断，或用户明确要求即时确认时，才暂停提问。
