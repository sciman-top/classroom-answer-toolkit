# 试卷参考答案交付规范-平台总则 v1.0

> 适用范围：面向中小学、多学科的试卷参考答案生成、校验、渲染与交付平台规则。
> 文件角色：本文件定义平台共性约束；具体学段与学科规则由 `commons/` 和 `subjects/` 补充。

## 1. 分层约定

1. 平台总则负责通用交付约束、目录约定、文件命名、渲染验收和资产同步规则。
2. `commons/` 负责跨多个学科或多个学段复用的规则。
3. `subjects/` 负责学段/学科专用源规范。
4. `compiled/` 负责对外转发和工具 `humanSpec` 所引用的完整规范产物。

## 2. 目录与命名

1. `样例交付/` 用于回归验证、演示和工具链示例。
2. `正式交付/` 用于真实生产交付。
3. `prompts/specs/` 是人类规范真值区，`docs/strategy/` 是规划与执行真值区。
4. 平台文件命名：`试卷参考答案交付规范-平台总则-v<version>.md`
5. 学段学科文件命名：`试卷参考答案交付规范-<学段学科>-源规范-v<version>.md`
6. compiled 文件命名：`试卷参考答案交付规范-<学段学科>-完整版-v<version>.md`

## 3. 交付基线

1. 默认最终交付物至少包含答案 `.md` 与答案 `.pdf`。
2. PDF 中数学公式必须走真实 LaTeX 数学渲染链。
3. 任何交付都应可追踪到使用的 subject-pack、snapshot 与人类规范版本。
4. 任何命名、目录或规范版本调整，都必须同步更新工具配置、测试样例和文档引用。

## 4. 人工复核与风险边界

1. 对高风险视觉内容，平台必须允许人工复核、抽检与降级。
2. 工具输出不得暴露内部审查过程、试错痕迹或模型自评。
3. 若证据不足，应标【疑】或进入复核，不得硬猜。
4. 若只是局部题图模糊或局部视觉理解不确定，但不阻断整份交付，应继续完成其余可确定部分，并在对应小问标注【疑】/待复核。
5. 只有当不确定性会阻断大范围求解、影响整份交付主结论，或用户明确要求实时确认时，才应暂停并提问。
