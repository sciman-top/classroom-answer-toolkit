> 产物类型：完整版。
> 汇编层：平台总则(v1.0)、K12通用(v1.0)、初中数学特异(v0.1)。
> 输出版本：v0.1。

## 平台总则

> 适用范围：面向中小学、多学科的试卷参考答案生成、校验、渲染与交付平台规则。
> 文件角色：本文件定义平台共性约束；具体学段与学科规则由 `commons/` 和 `subjects/` 补充。

## 1. 分层约定

1. 平台总则负责通用交付约束、目录约定、文件命名、渲染验收和资产同步规则。
2. `commons/` 负责跨多个学科或多个学段复用的规则。
3. `subjects/` 负责学段/学科专用源规范。
4. `compiled/` 负责对外转发和工具 `humanSpec` 所引用的完整规范产物。

## 2. 目录与命名

1. `样例交付/` 用于回归验证、演示和工具链示例。
2. `正式交付/` 用于真实生产交付。
3. `prompts/specs/` 是人类规范真值区，`docs/strategy/` 是规划与执行真值区。
4. 平台文件命名：`试卷参考答案交付规范-平台总则-v<version>.md`
5. 学段学科文件命名：`试卷参考答案交付规范-<学段学科>-源规范-v<version>.md`
6. compiled 文件命名：`试卷参考答案交付规范-<学段学科>-完整版-v<version>.md`

## 3. 交付基线

1. 默认最终交付物至少包含答案 `.md` 与答案 `.pdf`。
2. PDF 中数学公式必须走真实 LaTeX 数学渲染链。
3. 任何交付都应可追踪到使用的 subject-pack、snapshot 与人类规范版本。
4. 任何命名、目录或规范版本调整，都必须同步更新工具配置、测试样例和文档引用。
5. 默认主交付链不承诺 AI 自动基于题图生成答案图；若需插图，只支持用户提供或人工复核后的受控图块插入。

## 4. 人工复核与风险边界

1. 对高风险视觉内容，平台必须允许人工复核、抽检与降级。
2. 工具输出不得暴露内部审查过程、试错痕迹或模型自评。
3. 若证据不足，应标【疑】或进入复核，不得硬猜。
4. 若只是局部题图模糊或局部视觉理解不确定，但不阻断整份交付，应继续完成其余可确定部分，并在对应小问标注【疑】/待复核。
5. 只有当不确定性会阻断大范围求解、影响整份交付主结论，或用户明确要求实时确认时，才应暂停并提问。

## K12通用

> 适用范围：面向中小学各学段、各学科的试卷参考答案生成、校验、渲染与交付。
> 文件角色：承载跨学科、跨学段都普遍适用的规范。

## 1. 证据优先级

1. 原始 PDF、原始图片、原始截图是最高依据。
2. OCR、自动解析文本、上一轮答案、其他模型答案只能作为辅助。
3. 若原图与 OCR 冲突，以原图为准。
4. 若原图模糊、关键数据缺失、图表不可辨认，不得硬猜，必须标注【疑】。
5. 若疑点只影响局部小问，默认继续完成其他可确定部分，不因单点疑问中断整份交付。

## 2. 交付约束

1. 默认最终交付物至少包含答案 `.md` 与答案 `.pdf`。
2. PDF 中数学公式必须走真实 LaTeX 数学渲染链，不得降级为纯文本近似。
3. 最终输出不得暴露内部审查过程、试错痕迹、工具调用信息或模型自评。
4. 若存在疑点，应输出极简疑点清单，而不是在答案正文中混入长说明。
5. 若需要附图，默认仅允许插入用户提供或人工复核后的图块，不把 AI 自动看图作图作为通用承诺。

## 3. 目录与命名

1. `样例交付/` 用于演示、回归、冒烟测试，不与正式交付混放。
2. `正式交付/` 用于真实生产交付。
3. 人类规范采用“源规范 + compiled 生成物”模式，compiled 文件用于转发与工具引用。

## 4. 可读性与答题风格

1. 答案应优先服务课堂展示、打印分发与复核，不应写成冗长讲解稿。
2. 应优先输出结论、关键式、必要前提，不堆叠无新增价值的解释句。
3. 若用户明确要求补分析，应只补与结论直接相关的必要推导。

## 5. 人工复核原则

1. 高风险内容应允许进入人工复核，而不是强行全自动放行。
2. 复核结论应可追踪到具体证据，而不是停留在主观描述。
3. 仅当疑点会阻断整份交付的主要判断，或用户明确要求即时确认时，才暂停提问。

## 初中数学特异

> 适用范围：初中数学试卷参考答案生成、校验、渲染与交付。
> 文件角色：初中数学实验 subject-pack 的学科特异性源规范。
> 依赖层：平台总则、K12通用。
> 说明：本文件用于把 `math-answer` 纳入统一 human-spec 汇编链；当前只固定共享渲染/校验 contract 与少量数学格式约束，不宣称完整题型覆盖。

## 1. 当前范围

1. 当前 pack 的主要目标是验证平台 contract、snapshot、validator 与交付链路不依赖物理专有规则。
2. 当前自动化覆盖包括基础代数 smoke、分步等价变形 smoke、基础统计结论 smoke、基础概率记号 smoke、公式定界符硬失败和跨学科资产校验。
3. 几何作图、函数图像读图、图表驱动的统计/概率题和压轴综合题尚未建立专门规则，不得把当前 pack 视为生产级初中数学能力。

## 2. 数学特异性约束

1. 需要推导时，优先保留课堂友好的分步等价变形，不要压缩成只剩最终结果的一跳答案。
2. 公式、方程、分式、根号、上下标和单位必须走真实 LaTeX 渲染，不得把源码痕迹直接交付。
3. 对文本即可确定的数据列表基础统计题，可直接给出平均数、中位数、众数等标准结论，必要时可保留简洁计算式以便课堂展示。
4. 对文本即可确定样本空间与事件定义的基础概率题，可使用 `$P(A)=\frac{m}{n}$` 等标准记号，并保持事件名、分子分母含义和最终结论清晰。
5. 若题目依赖图形、坐标图、统计图表、实验过程或几何构造，而当前规则不足以稳定表达证据链，应进入人工复核或降级说明，不得伪装成已覆盖的通用能力。

## 3. 后续补强方向

1. 代数计算与方程组
2. 几何证明与作图
3. 函数图像与坐标系读图
4. 图表驱动的统计、复杂概率与应用题
